lab 29 Création d’un conflit

Buts

  • Créer un changement conflictuel dans la branche master.

Rebasculer sur master et créer un conflit 01

Revenir à la branche master et faire ce changement:

Execute:

git checkout master

File: lib/bonjour.rb

puts "Quelle est votre nom?"
my_name = gets.strip

puts "Bonjour, #{my_name}!"

Execute:

git add lib/bonjour.rb
git commit -m "Rendre interactif"

Voir les branches 02

Execute:

git hist --all

Output:

$ git hist --all
* 4075c2b 2014-02-21 | Rendre interactif (HEAD, master) [Jim Weirich]
| *   83299eb 2014-02-21 | Merge branch 'master' into saluer (saluer) [Jim Weirich]
| |\  
| |/  
|/|   
* | 5c4b651 2014-02-21 | Ajout d'un README [Jim Weirich]
| * 83b5dc6 2014-02-21 | Rakefile mis à jour [Jim Weirich]
| * 0560857 2014-02-21 | bonjour utiliser Faireunsalut [Jim Weirich]
| * 1e570bd 2014-02-21 | Ajout de la classe Faireunsalut [Jim Weirich]
|/  
* 6f69f38 2014-02-21 | Ajout d'un Rakefile [Jim Weirich]

* 80fc666 2014-02-21 | Using ARGV [Jim Weirich]

La branche master au commit “Ajout d’un README” a été fusionné dans la branche saluer, mais il y a maintenant un commit additionnel sur master qui n’a pas été fusionné dans saluer.

Et après 03

Les derniers changements dans master rentrent en conflit avec quelques changements existants dans saluer. Après, nous allons résoudre ces changements.
